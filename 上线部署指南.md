# 上线部署指南（让朋友从自己设备进来玩）

目标：把本地的游戏部署到网上，拿到一个公开网址（像 `https://你的项目.vercel.app`），谁都能打开。**全程不用命令行**，用 GitHub Desktop + Vercel 网页操作。免费。

---

## 准备：你需要两个免费账号
1. **GitHub** 账号 —— github.com 注册。
2. **Vercel** 账号 —— vercel.com，直接用 GitHub 登录最省事。

---

## 第一步：把项目传到 GitHub（用 GitHub Desktop，图形界面）

1. 下载安装 **GitHub Desktop**（desktop.github.com），用你的 GitHub 账号登录。
2. 菜单 **File → Add Local Repository**，选择文件夹 `D:\跑团\coc-trpg`。
   - 如果它提示"这里还不是仓库 / create a repository"，点 **Create a Repository** 确认即可。
3. 左下角填一句话（如 `first version`），点 **Commit to main**。
4. 右上角点 **Publish repository**。
   - **务必勾选 "Keep this code private"（私有）** —— 防止别人看到你的代码。
   - 点 Publish。

> 放心：项目里的 `.gitignore` 已经把含密钥的 `.env.local` 排除了，**密钥不会被上传到 GitHub**。密钥稍后填到 Vercel 里。

---

## 第二步：在 Vercel 导入并部署

1. 打开 **vercel.com**，用 GitHub 登录。
2. 点 **Add New… → Project**。
3. 在列表里找到刚发布的 `coc-trpg` 仓库，点 **Import**。
4. **关键：先别急着点 Deploy，先展开 "Environment Variables"（环境变量），把下面这些一条条加进去。**
   值就照抄你本地 `D:\跑团\coc-trpg\.env.local` 里的：

   | Name（名字） | Value（值，从你的 .env.local 复制） |
   |---|---|
   | NEXT_PUBLIC_SUPABASE_URL | https://xxxx.supabase.co |
   | NEXT_PUBLIC_SUPABASE_ANON_KEY | sb_publishable_... |
   | SUPABASE_SERVICE_ROLE_KEY | sb_secret_... |
   | LLM_PROVIDER | openai |
   | OPENAI_API_KEY | sk-svcacct-... |
   | LLM_MAIN_MODEL | gpt-4o |
   | LLM_AUX_MODEL | gpt-4o-mini |
   | IMAGE_PROVIDER | openai |
   | IMAGE_API_KEY | sk-svcacct-...（同 OpenAI key 即可） |

   每条：Name 填左列，Value 填右列，点 **Add**。全部加完。
5. 点 **Deploy**。等 1–3 分钟，看到庆祝画面就部署好了。
6. 点 **Visit / 你的域名**，会得到一个网址，例如 `https://coc-trpg-xxxx.vercel.app`。这就是你的公开网址。

---

## 第三步（可选但推荐）：回填站点网址

1. Vercel 项目页 → **Settings → Environment Variables**。
2. 新增一条：`NEXT_PUBLIC_SITE_URL` = 你的正式网址（如 `https://coc-trpg-xxxx.vercel.app`，结尾不要带斜杠）。
3. 去 **Deployments** 页，最近一次部署右边 **…→ Redeploy** 重新部署一次。

> 不填也能玩——邀请链接会自动用当前网址兜底。填了更稳妥。

---

## 第四步：和朋友一起玩

1. 把网址发给朋友。
2. 你打开网址 → 填代号 → **创建调查** → 进房间。
3. 房间里复制 **邀请链接**（带 `/join/` 的那条）发给朋友。
4. 朋友在他自己的设备打开邀请链接 → 自动以玩家 B 加入。
5. 选模组 → 各自建卡 → 正式跑团，两人实时联机。

---

## 以后改了代码怎么更新线上？
打开 GitHub Desktop → 填一句说明 → **Commit to main** → **Push origin**。Vercel 会**自动重新部署**，几分钟后线上就更新了。不用每次都重新配置。

---

## 几个要注意的
- **花钱**：线上每次 AI 回合、出图，都会真实扣你的 OpenAI 额度。公开网址意味着任何拿到链接的人行动也会扣你的钱——所以**别把网址到处公开发**，发给信任的朋友就好；并在 OpenAI 后台设个用量上限。
- **数据库共用**：线上和本地连的是**同一个 Supabase**，房间数据互通，正常。
- **匿名登录**：要确保 Supabase 里"Anonymous sign-ins"是开着的（你之前已开）。
- **没跑的迁移**：上线前确认 `migration_RUN_THIS.sql` 在 Supabase 跑过最新版（含 chat 修复、模组库、位置、视图权限修复）。
- **AI 结算偶尔较慢**：免费版函数最长 60 秒，案件生成/结算极少数情况下可能偏久，重试一次即可；要更稳可升级 Vercel Pro。
