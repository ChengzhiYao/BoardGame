# 持久世界模式 ·《列王纪 / Realms》· 设计文档 v0.1

> 工作代号 `realm`（名字可改：列王纪 / 群王之地 / 长河 / 九族…）。
> 一个**永久存在、所有人共享、永不重置**的活世界。每个玩家账号进入后只是这个世界里的
> **一个角色**；世界按真实世界 **10 倍流速**自动流动（真实 1 天 = 世界 10 天），
> 玩家的行动由 LLM 裁定其对世界的影响，但**不改变时间流速**。
>
> 本文先把架构、时间模型、世界引擎、数据模型、成本模型与分阶段落地定下来；
> 标 ❓ 的是需要你拍板的开放决策。落地时遵循《谜夜开发设计手册》里的既有约定
> （机制确定性、叙事 AI；真相/隐藏态 service_role-only；main/aux 双模型省钱；
> 结构化 JSON 契约；schema 增量进 `migration_RUN_THIS.sql`）。

---

## 1. 与现有模式的根本区别（为什么不能套用 room）

| 维度 | 现有模式（coc/dnd/…） | 持久世界 `realm` |
|---|---|---|
| 生命周期 | 开房→一局→结束即焚 | **永久**，永不重置 |
| 参与者 | 1 个房间 2 人 | **全体玩家共享同一个世界** |
| 时间 | 回合推进，无真实时钟 | **真实时钟 ×10 自动流动**，无人在线也在走 |
| 状态规模 | 一局的隐藏真相 + 几十条线索 | 庞大世界 canon + 持续演化的活体状态 |
| 入口 | "开房 / 加入" | "**进入世界 / 创建你的角色**" |
| 前端 | `app/room/[id]/` | 需新顶层 `app/realm/`（不是 room） |
| 计费 | 买"局数"，开房扣 1 局 | ❓ 局数模型不适用，需新计费（见 §9） |

**结论**：`realm` 复用底层能力（Supabase/LLM 适配器/图片风格锁/i18n/计费框架/cron），但要新建**世界引擎**、**时间系统**、**世界数据层**和**独立前端入口**。它不挂在 `rooms` 上。

## 2. 世界观结构（庞大 canon · 先建后跑）

世界的"庞大"靠**一份预先生成并锁定的世界圣经（World Bible）** + **持续演化的活体状态**两层实现。canon 是地基，活体状态是地基上的流动。

### 2.1 World Bible（canon，一次性生成 + 人工校订后锁定）
类似 CoC 的"隐藏案件档案"，但规模大得多、且**大部分公开**（玩家需要了解世界）。内容：
- **地理**：3 个王国，各自的疆域、首都、关键城邦、地形、气候、资源带、边境与争议地。
- **种族体系（出身随机 · 带稀有度 · 10 族）**：出生按权重随机掷定 10 族之一 → 人类 35 / 兽人 22 / 精灵 14 / 矮人 9 / 龙裔 5 / 鬼族 4 / 妖精 3 / 海族 2 / 灰触者 1 / **魔族 5**（基准；详见《世界圣经》§2、§8A）。**魔族是世界的共敌**，生为魔族 = 最硬核的敌对出身（魔界/潜伏人间、被各势力追猎、可走晋升/挑战/反叛魔王路线）；**黑暗作答可经亲和把魔族概率推高到约 30–40%**。各族有分布、属性/天赋倾向、寿命与地位；测试给各族**亲和偏置**影响出身（§5C）。
- **8–10 个宗族（Houses）**：族名、纹章/族徽、封地与族城、族性（尚武/经商/信仰/学术…）、家训、当前族长与继承线、**公开关系**（盟友/世仇）与**隐藏议程**（service_role-only）。
- **创世神话 + 悠久编年史**：创世神话（世界与诸族的起源）→ 建国 → 王国演变 → 几次大战/王朝更替/灾变 → 走到"当下"。这条历史是**已发生的固定 canon**，写成可被引用的编年史，玩家入场即站在历史的"现在"。
- **神明与信仰（各国各族不同）**：每个王国/种族有自己的神系与信仰、神职/教团；信仰差异本身就是世界张力来源。另含商会、骑士团、暗影组织等跨宗族 factions。
- **语言与文化习俗**：各国/各族不同的语言（及一门通用语）、礼俗、节庆、婚丧、禁忌——影响角色出身带来的文化背景与他人对其的观感。
- **当下张力（开场局势）**：埋好 2–4 条**大势**（一场酝酿中的继承危机、一道封印松动的古老威胁、一条干旱/瘟疫线、一桩王室联姻），作为世界开场就在走的"世界时钟"。

> canon 生成是**一次性大成本**（多次 LLM 调用 + 人工润色），生成后写入 `world_lore` 并 `locked`，之后只读、可被活体状态引用，**不再重复生成**——这正是你要的"不会每次打开就重新生成"。

### 2.2 活体状态（mutable，随时间与玩家流动）
- 宗族的**资源/影响力/兵力/声望**、彼此**关系值**。
- 各**地区的控制权、民心/动荡值、物价**。
- 进行中的**世界剧情线（world clocks）**：战争进度、继承推进、瘟疫蔓延、预言临近…到点触发后果。
- 重要 **NPC** 的状态（生老病死、升迁、阴谋）。
- **编年史（Chronicle）**：世界发生过的事件流水账，玩家可读的"历史正在被书写"。

## 3. 时间模型（真实 ×10，自动流动）

### 3.1 连续时钟（零成本、确定性）
游戏时间 = 公式计算，**不需要逐刻存储**：

```
game_time = world_epoch + (real_now − real_epoch) × 10
```

- 真实 1 小时 = 世界 10 小时；真实 1 天 = 世界 10 天；真实 1 周 ≈ 世界 2.3 个月；真实 1 月 ≈ 世界 10 个月；**真实 1 年 ≈ 世界 10 年**。
- 历史推进很快：真实几个月，世界就走过一代人——支撑"王国演变、宗族兴衰"的史诗感。
- 前端随时把它渲染成世界历法（如"长河历 318 年·暮春·三月十二"）。`lib/realm/time.ts` 提供换算与历法格式化。玩家行动**不改变**这个流速（按你的要求）。

### 3.2 世界心跳（Heartbeat · 模拟步进）
时钟在连续走，但"世界里真的发生事"需要一个**定时模拟步**。采用**混合**（你的选择）：

- **主心跳 = 每真实日 1 次（cron）**：推进一个"世界篇章"（≈10 游戏天）。处理：到点的 world clocks、宗族按议程行动、资源/民心结算、聚合这段时间玩家行动的**区域级/王国级**涟漪、生成 1–3 条重大事件并由 LLM 叙述、写入编年史。**成本由"宗族/势力数量"决定，与在线玩家数无关** → 加人不爆成本。
- **可选轻心跳 = 每真实小时（cron）**：只跑便宜的确定性微更新（物价波动、局部小事件、推进快速 clocks），基本不调 LLM。
- **玩家行动即时结算**（见 §4）：介于心跳之间，玩家的具体行动当场被 LLM 裁定局部后果，立即落库——这就是"混合"里玩家驱动的那一半。

> 心跳路由 `/api/realm/tick` 由 `vercel.json` cron 触发，并用 `CRON_SECRET` 校验，禁止外部乱调。

## 4. 世界引擎（分层：确定性模拟 + LLM 裁定）

遵循项目"**机制确定性、叙事 AI**"原则，把贵的 LLM 用在刀刃上。

### 4.1 确定性模拟（`lib/realm/sim.ts`，纯函数，跑在心跳里）
代码规则推进可量化的东西：宗族影响力/资源/兵力的增减、地区民心与动荡、战争/围城进度、物价、world clocks 倒计时与到点触发、NPC 的程式化行动。**几乎零 LLM 成本**，可复现、可调参。类比并放大 CoC 的 `world_clock` + suspicion 机制到"宗族/王国"尺度。

### 4.2 LLM 裁定（`lib/realm/prompt.ts`，结构化 JSON，少量调用）
LLM 只做四类事：
1. **世界圣经生成**（一次性，genesis）。
2. **玩家行动裁定**：读玩家行动 + 其角色处境 + 相关世界切片 → 判定**结果与影响层级**（见 §5），输出结构化 JSON（叙事 + 状态变更 + 影响标记 + 是否上抛到区域/王国级）。
3. **重大事件叙述**（心跳里，把确定性模拟算出的大事写成有画面感的编年史条目）。
4. **关键 NPC 对话/阴谋**（按需，节制）。

模型分层：解析/小事用 `aux`（便宜），有后果的裁定与叙述用 `main`。上下文**只喂相关切片**（该角色所在地区、所属宗族、最近编年史、被 `pgvector` 检索到的相关 lore），绝不把整本圣经塞进去。

## 5. 玩家行动与"影响世界"的分层（关键设计）

要同时满足"每个决定都影响世界" + "一个人别轻易改写庞大世界" + "成本可控",用**影响力分层 + 行动预算**：

### 5.1 三级涟漪
- **局部（local）**：绝大多数行动——交涉、打探、交易、决斗、营建、探索。**当场由 LLM/规则结算**，只影响该角色与身边人/地，立即写编年史的局部条目。便宜、即时。
- **区域/宗族（regional）**：有分量的行动累积——一连串阴谋削弱某宗族、占据要地、煽动民变。这些**改的是活体状态的标量**（影响力/民心/关系值），在**心跳时聚合**成区域级变化。
- **王国/世界（world）**：极少数高风险、常需**多人协作**的行动才能撬动（弑君、改写继承、解放/释放古老威胁、终结一场战争）。这类在心跳由模拟 + LLM 共同裁定，会进入世界级编年史与历史。

> 这样"每个决定都算数"（都进编年史、都微调状态），但**世界级走向需要积累或协作**，既真实又防滥用、又控成本。可给每个影响层级设**封顶**（单个玩家单位时间对宗族影响力的最大改变量）。

### 5.2 行动节奏（主闸门 = 时间占用，见 §5D）
- **主限制**：一次只能做一件事，每个行动按真实时间倒计时占用（§5D）——这天然防刷、压成本、制造"决策有分量"。
- 可选叠加一个很宽松的每日重大行动**辅助上限**防极端刷子；多数玩家碰不到。具体 ❓ 待定。

### 5.3 行动流程
```
玩家提交行动 → 入队 player_actions(pending)
  → 服务端取相关世界切片 + 角色状态
  → aux 解析意图/是否需要判定/影响层级预估
  → 规则/LLM 裁定结果（local 即时；regional/world 标记上抛）
  → 加区域/对象级 advisory lock，原子写活体状态 + 编年史
  → Realtime 广播 → 同区域玩家看到世界在动
```

## 5A. 角色生命周期 · 一号一命（随机出身 · 童年起步 · 永久死亡）

- **账号绑定**：每个 Gmail 账号 = 世界里**唯一固定角色**。**必须 Google 登录**（匿名访客不适用 `realm`——要绑定 gmail 身份，也与计费一致）。
- **随机出身**：进入世界时**完全随机**掷定 → 种族 + 出生宗族/家庭 + 出生地，**不可挑选**（命运随机是这个世界的灵魂）。出身可能是某宗族旁支、平民、边地流民…决定起点资源与社会位置。
- **童年起步**：角色从**孩童**阶段开始，随时间长大；童年期的经历/关键抉择塑造初始属性与性格倾向。
- **永久死亡（permadeath）**：角色会衰老、受伤、被杀、病死。**死亡后该账号才能重新创建**一个新的随机角色，重新从孩童开始。一条命就是一段人生。

**时间尺度 · 已定 = C（童年期局部加速）**：未成年阶段用**更高倍率**让童年成长所用的真实时间更短，成年后回落到标准 ×10。
- 实现：`lib/realm/time.ts` 的年龄推进对 `life_stage ∈ {child, youth}` 套一个 `growth_multiplier`（如童年再 ×N），成年后回归 ×10。世界历法时钟本身仍是统一 ×10（不受影响）；被加速的只有**该角色自身的年龄/成长进度**。
- 待定的只剩具体倍率/各阶段时长 ❓（如童年压缩到几真实天/周成年），可在阶段 1 调参。

## 5B. 情报差 · 感知层（玩家只见角色视角）

世界状态在服务端**完整且权威**，但玩家**只能透过自己角色的感知**看世界——这是项目"私人线索/情报差"DNA 放大到世界尺度。

- **玩家看不到**：世界全貌、其它地区正在发生的事、宗族的隐藏议程、远方战争的细节、别人的私密行动。
- **玩家能看到**：自己亲历/亲见的事、所在地与到访过的地方、认识的人、**传到耳边的消息**（可能延迟、可能失真的传闻）。
- **情报如何传播**：重大事件在心跳/玩家移动/社交接触时，按**距离与社会网络**向相关角色"扩散"成知识条目——近的快而准，远的慢而易走样。一个远方王国的政变，可能数（游戏）周后才以一条不确定的传闻抵达你。
- **实现**：全局真相写 `realm_chronicle`（权威、service-only）；每个角色一张**知识表** `realm_knowledge`，记录"它知道什么、怎么知道的、准不准、何时知道"。**前端只读自己的知识表 + 全民常识级 lore**，绝不直查全局编年史。RLS 物理隔离，与 CoC 的 `visible_to`/`my_seat` 同源，只是粒度细到"每个角色"。

## 5C. 开局人格测试 → 角色创生（24 题 · 一次性 · 可跳过）

新玩家进入世界的第一步：先做人格测试，再降生为孩童。

### 流程
```
人格测试(24题) → 确定性算分 → [LLM]性格解读结果页 → 随机掷出身(种族/家庭/出生地)
  → 用测试塑造这个孩子的天赋/属性/性格/数值 → 以"孩童"降生 → 进入童年(局部加速·§5A)
```
- **命（随机）+ 性（测试）**：家庭/出生地按稀有度**纯随机**；**种族**在 **9 族**中按稀有度随机、并由测试的**亲和偏置**加权（你的天性让某些族更可能，但不完全决定）。人格测试还决定这个孩子的**灵魂**——天赋、擅长、性格与各项数值。命运随机 ＋ 天性加权 ＝ "你成为的那个具体孩子"。

### 规则
- **24 题**，每题一个情境/抉择，每个选项**预设权重**，加到下面的多维评分上——**确定性算分、零 LLM 成本**。
- **可跳过**：跳过的题按**概率分布**自动补值（可按已答倾向/出身轻微加权），不阻断流程。
- **一次性不可重测**：测试与该角色绑定，做完即锁（`test_locked`）。**唯有角色死亡、重建新角色时才会重新测试**。
- **结果页（一次性揭晓）**：算分后由 LLM 生成一段**性格解读**——"你是一个怎样的人 + 为什么这么判断（引用你的关键作答）"，连同天赋/擅长一并展示，给玩家"被看穿"的仪式感。仅此一次 LLM 调用（main），其余全确定性。

### 角色数值模型（分层 · 庞大世界需要的全套）

> **关键设计**：24 题测试只播下"**灵魂内核**"（A–D）；其余各层来自**随机出身**（E–G）与**一生经历**（H–J）——别让测试背负所有维度（24 题解析不了太多）。这样既撑得起庞大世界，又保证测试有效。

**▶ 测试播种（test-seeded · 24 题决定）**
- **A. 核心属性 Attributes（1–100，d100 判定底）**：体魄 / **体质** / 敏捷 / 心智 / 魅力 / 意志 / 感知（＋命运·运）。〔7 项＋运；从"体魄"拆出**体质**以支撑老化/伤病/耐力〕
- **B. 天赋·性向 Aptitudes（领域成长倍率）**：武勇 / **治政** / 谋略 / 学识 / **信仰** / **秘法** / 交际 / **百艺生存**。〔8 项；信仰与秘法分家、治政独立——对应权谋世界的核心支柱。每项 = 该领域技能的**起始加成＋成长倍率**，再叠**种族修正**〕
- **C. 性格 Personality（对立轴）**：勇毅↔谨慎 ｜ 仁善↔冷酷 ｜ 正直↔权诈 ｜ 理智↔激情 ｜ 野望↔安分 ｜ 独行↔合群 ｜ 虔信↔怀疑 ｜ **慷慨↔贪婪** ｜ **忠贞↔背离**。〔9 轴；影响 NPC 反应与 LLM 如何演绎后果〕
- **D. 起始特性 Traits（离散 · 1~2 个）**：测试＋种族赋予的天赋异禀/缺陷——过目不忘 / 天生领袖 / 体弱多病 / 龙息〔龙裔〕/ 灰变〔灰触者〕/ 美貌…。离散特性给世界更多质感。

**▶ 出身播种（birth-seeded · 随机掷定）**
- **E. 身份·地位 Standing**：阶层（贵族/自由民/农奴/奴隶/化外）、家族、头衔、财富、封地产业、初始声望。
- **F. 信仰 Faith**：所属神系＋虔诚度（来自出身文化，日后可改宗）。
- **G. 语言 Languages**：出身带来的母语/通语（接入情报差 §5B）。

**▶ 一生累积（life-grown · 玩出来的）**
- **H. 技能 Skills（具体 · 可学可长）**：剑术/弓术/骑术/谈判/欺瞒/医术/读写/锻造/航海/狩猎/祭祀/咒法/各国语言… **成功率 = 相关属性＋技能 − 难度（d100，仿 CoC）**；**成长 = 基础 × 天赋倍率 × 年龄阶段**（童年学得最快·§5A）。测试只给少数**起始技能倾向**。
- **I. 状态 Condition**：健康/伤病、心境·精神（创伤/压力，SAN 式）、名望（荣誉↔恶名）、年龄/生命阶段。
- **J. 关系网 Relationships**：与 NPC/玩家的纽带（血亲/姻亲/忠诚/世仇/恩债）——活体政治世界的关键。

**判定法**：d100，`掷骰 ≤ 相关属性 + 技能 − 难度修正` 即成功（沿用 CoC 成功等级：大成功/极难/困难/普通/失败/大失败）。重大/模糊的行动再交 LLM 据数值裁定叙事。

### 样题（示意 3/24 · 确认维度后我再写全 24 题及权重表）
1. 村里失火，你先冲去救……？　A 邻家的孩子（仁善+、勇毅+）／B 自家的口粮（安分+、谨慎+）／C 趁乱去看看里正家的箱子（权诈+、谋略+）／跳过
2. 长者讲古时，你最入神的是……？　A 英雄战死的段落（武勇+、激情+）／B 智者设局翻盘（谋略+、心智+）／C 神明显灵的传说（虔信+、信仰+）／跳过
3. 做错事即将被发现，你会……？　A 抢先坦白（正直+、勇毅+）／B 想个说法圆过去（权诈+、魅力+）／C 一声不吭扛下（意志+、独行+）／跳过

## 5D. 行动时间占用 · 一次只做一件事（真实世界节奏 · 核心节奏）

这是个**真实世界**：做任何事都要花时间。玩家发起一个行动后进入**倒计时占用**——在它完成前，角色处于"**忙碌**"，**不能再发起别的行动**；倒计时走完、行动结算，才能做下一件事。

- **每个行动有时长（游戏时间）**，按 ×10 折成**真实倒计时**（§3）。参考：1 游戏时辰 ≈ 6 真实分钟；1 游戏日 ≈ 2.4 真实小时；1 游戏周 ≈ 17 真实小时。所以"赶三天路" ≈ 7 真实小时占用——这是一款**慢节奏、可挂机、定期回来查看**的持久人生游戏（类 Fallen London 行动经济 ＋ 挂机 MMO 的真实计时）。
- **时长由行动类型 ＋ 数值决定**：移动按距离/坐骑、学技按熟练度与天赋、锻造按工序、疗伤按伤势与体质。**高敏捷赶路更快、高百艺打造更快**——数值不只影响成功率，也影响**耗时**。
- **忙碌期间只能做被动的事**：查看世界、读编年史/情报、草拟要说的话（待空闲再发）；**不能发起新的 committed 行动**（大部分时间"动弹不得"，符合真实）。
- **离线也在走（关键）**：倒计时是真实时间，**关掉浏览器也继续**。回来时若已到点，服务端**惰性结算**（或由心跳替离线玩家结算）已完成的行动——这正是"永久世界"该有的样子。
- **可被打断（真实感）**：行进中遭袭、家中失火等突发事件可**中断占用**，强制进入应对——世界会插手你的计划。
- **主动放弃（可中途取消）**：玩家可在倒计时结束前**主动取消**当前行动，立刻恢复自由。代价按真实结算：
  - **已花的（游戏）时间不退**（时间照常流逝，真实世界不能倒带）。
  - **按进度结算**：移动→停在路途中间的某处（已走的算数）；学技/锻造/调查→进度部分保留或作废（视类型，可能浪费已耗材料/精力）；社交/密谋→半途中断可能留下后果（NPC 觉察、话只说一半被误解）。
  - **无全额收益、但不重罚**：放弃 = 拿不到完成奖励，这是正当选择，重真实而非惩罚。
  - 少数**有"不归点"**的行动例外（已交手的决斗、已发动的刺杀等），会明确标注不可取消。
- **取代"每日行动预算"**：节奏主要由"一次一事 ＋ 真实耗时"自然限制（也顺带压住 LLM 成本），§5.2 的每日配额降为**辅助上限**或取消。

❓待定：①是否允许**预约下一件事**（idle 式排队，set-and-forget，完成自动接上）——推荐留 1 个排队位提升挂机体验；②突发中断的烈度（哪些事能打断、打断惩罚）；③各类行动的**基准时长表**（移动/训练/打造/社交/调查/疗伤…），落地时定一张可调表。

## 5E. 经济与活动系统（货币 · 资金消耗 · 打工 · 冒险工会 · 冒险）

经济是这个真实世界的骨架：**赚（打工/冒险）→ 花（生活/装备/训练/打点）→ 驱动野心、阶层流动、甚至生死**。经济计算全**确定性**（零/极少 LLM），只有冒险的关键叙事用 LLM。注意：**游戏内金币 ≠ 真实付费**（$30 解锁是真实货币，与游戏经济互不相通）。

### 货币
- 三级硬币：**铜钿 / 银元 / 金冠**（1 金 = 100 银 = 10000 铜，可调）。财富计入角色 Standing（§5C·E）的 `wealth`。
- 各王国可有自己的铸币（卡斯泰兰金冠 / 朝雾两 / 北境多以物易物）；跨国经**钱庄兑换**收手续费——汇差即商机，也是情报差的一环。起步先用统一三级币，后续再加分国币。

### 资金消耗（让钱有意义的"沉"）
- **生活开销（按心跳周期扣）**：食宿、税赋按**生活水准 lifestyle**（贫民/平民/体面/贵族）周期性从 wealth 扣；水准越高越费钱，但带声望与机会。
- **其它支出**：购置/修缮装备、拜师学技的束脩、疗伤问医、贿赂打点、租房置产、雇人。
- **付不起的后果**：欠债→利滚利、被讨债；断粮→饥饿掉体质/健康（§5C·I）；交不起租→被逐流落。**穷会真的要命**——这是 permadeath 世界的压力源之一。

### 打工系统（低风险 · 稳收入 · 慢成长）
- 打工是一种**时间占用行动**（§5D）：去做工，倒计时结束领**工钱**并**缓慢提升相关技能**。
- 岗位按**地点/阶层/技能**开放：农工、码头扛活、酒馆跑堂、铁匠学徒、抄书吏、护院、商队脚夫…
- 工钱 = 基础 × 技能/天赋系数 × 时长。打工**安全但封顶低**，是冒险破产时的活路、平民起步的正道。

### 冒险工会（institution · 跨国 faction）
- 城镇设**冒险者工会**（一个跨王国势力，呼应异世界感）。玩家可**入会**，凭完成委托积累**工会等级/声望**，解锁更高阶委托与权益（情报、补给折扣、通行护照、抚恤保险）。
- 工会等级 = 面向工会这个 faction 的**声望/地位**（接 §5C·E 与关系网 §5C·J）。

### 冒险系统（高风险 · 高回报 · 接世界）
- **委托来源**：工会布告板、NPC 私托、世界自发事件。**类型**：讨伐魔物/盗匪、护送、采集、探索遗迹、递送、调查、平乱。
- **机制**：每个委托是一次**时间占用行动**，过程用**确定性 d100 判定/战斗引擎**结算（属性＋技能−难度）；**奖励** = 金钱＋物品＋工会声望＋技能成长；**风险** = 受伤/残疾/**死亡（permadeath！）**。
- **接活体世界**：委托与世界自洽——"清剿盗匪"的盗匪可能是某宗族爪牙；放着不管，对应 world clock 会推进（村子被毁、势力坐大）。重大委托用 LLM 叙述并写编年史；普通委托走**模板池**（便宜）。
- **冒险 vs 打工**：稳而低 vs 险而高——玩家自择活法，通往不同人生轨迹与结局。

## 6. 数据模型（Supabase · 新增表，进 `migration_RUN_THIS.sql`）

按 §3 安全红线设 RLS：canon 公开只读；隐藏议程/私密状态 service_role-only；编年史按可见域分级；玩家只能写自己的 pending 行动。需要前端实时刷新的表加进 `supabase_realtime` publication。

- `realm_worlds`：世界（单一共享世界，或少量"纪元/分线"shard）。`epoch_real`、`world_epoch`(游戏起始历法)、`time_scale=10`、`season`、全局 flags。
- `realm_lore`：世界圣经条目。`kind`(geo/house/history/religion/faction/current)、`title`、`body`、`is_public`、`embedding vector(1536)`、`locked`。（隐藏议程类 `is_public=false`，service_role-only。）
- `realm_houses`：8–10 宗族。族名、纹章、封地、族性、`resources jsonb`、`influence`、`relations jsonb`、`secret_agenda`(机密)、当前族长 NPC 引用。
- `realm_regions`：地区。所属王国、`controlling_house`、地形、资源、`unrest`、物价。
- `realm_factions`：跨宗族势力（教团/商会/暗影）、议程、势力值。
- `realm_characters`：角色。`user_id`(NPC 为 null)、`gmail`(玩家唯一绑定)、姓名、`race`、`house_id`/`region_id`、`birthplace`、`birth_game_time`、`life_stage`(child/youth/adult/elder)、`stats jsonb`、声望、`holdings jsonb`、`status`(alive/wounded/dead)、`death_game_time`、`is_npc`、`private_state`(机密，service-only)。**每 Gmail 账号唯一角色；permadeath 后才能重建新随机角色。** 用 `unique(gmail) where is_npc=false` 保证一号一命。
- `realm_chronicle`：编年史/事件流（**全局权威真相，service-only**，玩家不直读）。`game_time`、`scope`(local/region/kingdom/world)、`title`、`narration`、`cause`(player/sim)、`effects jsonb`、`origin_region`。
- `realm_knowledge`：**角色知识/情报表（玩家读世界的主入口）**。`character_id`、`ref`(指向某 chronicle 事件 / lore / 人物)、`how`(witnessed/told/rumor)、`accuracy`(true/distorted/false)、`confidence`、`learned_game_time`、`content`(该角色所知的版本，可能与真相不符)。RLS：玩家只能 select `character_id` 属于自己的行。
- `realm_clocks`：世界时钟。`due_game_time`、`hidden`、`on_fire`、`fired`——持久化、全局化的 CoC world_clock。
- `realm_characters` **增占用字段（§5D）**：`busy_until timestamptz`(忙碌至·真实时间)、`current_action jsonb`(当前行动:type/started_at/ends_at/game_duration/state)。`now < busy_until` 即忙碌、不可发起新行动。
- `realm_actions`：玩家行动队列。`character_id`、`content`、`status`(queued/active/resolving/resolved/cancelled)、`submitted_real`、`started_at`、`ends_at`(真实·倒计时终点)、`game_duration`、`tier`(local/regional/world)、`result jsonb`、`cost`。**结算 = 惰性(玩家回来/轮询时) + 心跳兜底(离线玩家)**。
- `realm_characters` **增数值字段（分层模型 §5C）**：`attributes jsonb`(A)、`aptitudes jsonb`(B)、`personality jsonb`(C)、`traits jsonb`(D 特性)、`growth_rates jsonb`、`standing jsonb`(E 阶层/头衔/财富/封地/声望)、`faith jsonb`(F)、`languages jsonb`(G)、`skills jsonb`(H 具体技能与熟练度)、`condition jsonb`(I 健康/精神/名望)、`test_locked bool`。
- `realm_relationships`：关系网(J)。`from_char`、`to_ref`(NPC/玩家)、`kind`(血亲/姻亲/忠诚/世仇/恩债)、`value`、`note`。
- `realm_origin_tests`：开局人格测试。`character_id`、`answers jsonb`(含跳过标记)、`scores jsonb`、`result_text`(性格解读)、`created_at`。一角色一行，锁定不可改。
- **经济与活动（§5E）**：角色 `standing.wealth`{gold,silver,copper}、`standing.lifestyle`、`standing.debts`。
  - `realm_quests`：委托。`region_id`、`source`(guild/npc/world)、`type`、`rank_req`、`risk`、`game_duration`、`rewards jsonb`、`status`(open/taken/done/failed/expired)、`taken_by`、`linked_clock`(可空·接 world clock)。
  - `realm_guild_membership`：`character_id`、`guild_faction_id`、`rank`、`renown`、`quests_done`。
  - `realm_ledger`(可选)：交易流水（赚/花/欠），审计与防刷；打工岗位可作**配置**(地点×岗位)而非表。冒险者工会作为一个 faction 写入 `realm_factions`。
- 复用：`api_usage`（成本）、`images`（出图，走风格锁）、`error_logs`。

## 8. 成本模型（能不能扛住的核心）

- **背景成本与玩家数解耦**：心跳成本由宗族/势力/事件数决定（个位数到几十），不随在线人数增长。每日主心跳 ≈ 少量 main 调用（叙述几条大事）+ 大量零成本确定性模拟。

## 10. 分阶段落地（MVP 优先，控风险）

- **阶段 0 · 创世（Genesis）**：用 LLM + 你的审定，生成并**锁定**世界圣经（3 王国、8–10 宗族、历史时间线、开场局势）。产出 admin 工具 `/api/realm/genesis`。这步定了，世界就"永久存在"了。
- **阶段 1 · 活世界只读**：×10 时钟 + 世界总览 + 编年史；玩家可**进入并创建角色**、浏览世界；每日心跳推进历法并由确定性模拟 + LLM 叙述 1–3 条大事。先让世界"会呼吸"。
- **阶段 2 · 玩家行动（local）**：提交行动 → LLM 裁定局部后果 → 写世界 + 编年史；行动预算/冷却。让"决定影响世界"在局部成立。
- **阶段 2.5 · 经济与活动**：货币 / 资金消耗 / 打工（全确定性）先行，再上冒险者工会与委托（确定性判定 + LLM 叙述）。给世界注入"过日子"的经济循环——这是玩家日常的主要内容。
- **阶段 3 · 宗族模拟 + 涟漪聚合**：心跳里宗族按议程行动、战争/继承推进；玩家行动聚合成区域/王国级变化。世界开始"自己演变"。
- **阶段 4 · 多人交互与深度**：同区域/同族玩家结盟、背叛、争权；世界级行动需协作；出图、NPC 阴谋、计费与审核完善。

> 建议先一起敲定 §9 的开放决策，然后从**阶段 0 创世**做起——先把那个"庞大世界"建出来锁住，后面所有玩法都长在它上面。
